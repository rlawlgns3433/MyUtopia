Introduction

Isle of Lore 2: RPG Item Icons is a pack of 200 icons covering all your RPG, adventure, roguelike and strategy game needs, with icons for resources, equipment, weapons, armor, food and more. They work great with my other Isle of Lore 2 packs, including Isle of Lore 2: Hex Tiles!

The icons come as png files with and without a white outline and also as Krita (.kra) and Photoshop (.psd) source files for your personal edits. The pack is compatible with my batch image editing tool Color Mapper, which means you can easily change the color palette of the icons to your own liking... the required Color Mapper Files are included!

This documentation will go into more detail on how to find the files you want and how to change and re-export them to fit your needs.

Please keep the license in mind, which is available as a License.txt-file in the folder you downloaded. If you have questions or feedback, you can reach me through the following channels. Content Structure

On the top level, the asset pack contains the folders Assets, Documentation, Scenes and Sources.

If you want to use the assets as they are, head over to the Assets-folder and use the files in there. The folder Scenes provides a very basic but exemplary setup.

The icons come in two variants. The first variant consists of the icons with a white outline. The second variant has no outline. Both variants have their respective folders rpg_item_icons.standard_outline and status_icons.standard_transparent within the Assets-folder.

In case you want to edit the assets, unpack the Sources.zip in the Sources-folder. There you find 3 sub-folders called input, output and misc, a folder which contains a list of the icon names written out, sorted by ID. The output-folder is empty, as these files were already moved to the Assets folder we discussed earlier. The input-folder though contains the source files, including Krita (.kra) and Photoshop (.psd) files from which the images were created as well as single png-exports of all the layers, which can be used with my tool Color Mapper to setup a regeneration of all the assets, which allows you to change colors and other properties of the assets automatically. A dedicated section further down will go into more detail. The Color Mapper Files standard_outline.colormapper.txt (etc.) tell Color Mapper how to assemble the icons.

Re-exporting the Icons with Color Mapper

Please make sure to use the most up-to-date version of Color Mapper, as this asset pack requires the two new colors wood and wood_dark to be added to the config.

If you want to use the icons the way they are, refer to the Sources\output-folders. But if you want to change the icons, for example replace some or all of the colors, you can use my batch image editing tool Color Mapper. It can also re-size, re-scale and re-position icons, all in one go.

Color Mapper assembling an icon from its single parts.

Color Mapper assembling an icon, but also switching out some of the colors.

To make Color Mapper work, the Isle of Lore 2: RPG Item Icons download includes the folders Sources\input\rpg_item_icons, containing the Krita (.kra) and Photoshop (.psd) source files as well as png exports of the single layers and Color Mapper Files (.colormapper.txt) containing information on how Color Mapper is going to assemble the files.

Color Mapper isn't part of this asset pack and instead has to be downloaded here, which allows me to release updates for Color Mapper independently of the projects it is used in. The download includes a Documentation-folder with exact instructions on how to use Color Mapper, but the workflow in general is at follows:

  1. Setup Color Mapper's configuration file so it knows where to find the Sources\input and Sources\output folders.
  2. Running Color Mapper, which will gather all the *.colormapper.txt-files in the input folder and assemble the png files from the single layer files with colors referenced in the *.colormapper.txt-file and defined in Color Mapper's configuration file. The assembled png files will be put in the output-folder.
  3. After checking the results, the files from the output-folder are ready to use.

Make sure to keep the naming scheme rpg_item_icon_(name)_(id)_(layer).png intact, for example rpg_item_icon_wood_0_line.png for the line-layer of the Wood icon which has the id 0.
